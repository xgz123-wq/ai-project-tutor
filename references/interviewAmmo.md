## 面试弹药库（把"懂"变成"讲得出"）

自学者最大的坑：看懂了，但面试一问就卡。本文件解决"理解 → 能讲"的最后一公里。
讲完每个模块后，主动产出下面的「三件套」。

---

### 面试弹药三件套（每个模块都产出）

```
## 模块：{模块名}

### 🎤 面试官可能怎么问
- {结合该模块的设计取舍，列 2-3 个真实会问的问题}
  例：你这个 RAG 召回不到题目怎么办？

### 💬 你可以怎么答（STAR 结构）
- 背景(S)：{这块要解决什么问题}
- 任务(T)：{具体要做到什么}
- 行动(A)：{你/项目是怎么做的，讲清关键技术决策}
- 结果(R)：{效果，能量化更好}

### 🧠 为什么这么设计（讲清 tradeoff，这是区分"会用"和"懂"的关键）
- 为什么选 A 不选 B：{...}
- 不这么做会怎样：{...}
```

### 这个项目的高频面试问题清单（按模块）

**简历解析**
- PDF 文本怎么提取的？解析成结构化失败了怎么办？
- 为什么解析用低温度（temperature 低）？
- 实习生和社招的简历评分标准为什么要分开处理？

**岗位匹配 Agent**
- 为什么用 Agent（tool-calling）而不是一个大 Prompt 全搞定？
- 5 个工具是怎么串起来的？谁决定调用顺序？
- 工具的 docstring 为什么重要？写不好会怎样？
- LangChain 的 `@tool` 接不了 FastAPI 的 Depends，DB session 怎么管的？
- `max_iterations` 是干嘛的？不设会怎样？
- Agent 中间步骤为什么要做摘要再返回？

**RAG 出题**
- 整条 RAG 链路讲一遍（向量化→存储→召回→生成）？
- 为什么用 pgvector 而不是 Faiss/独立向量库？
- 召回不足时的兜底策略是什么？（二次召回→AI 补全→纯 AI 生成）
- 为什么把题目和参考答案拼一起再向量化？

**AI 评分**
- 为什么不让大模型自由打分，而要注入 reference_answer 和 key_points？
- 流式评分（SSE）在体验上解决了什么？
- 评分结果怎么保证稳定、可解释？

**结构化输出（跨模块高频）**
- 大模型不吐纯 JSON 怎么办？（讲 `_extract_json` 的三级兜底）
- 为什么要兜底而不是直接信任模型输出？

**工程化**
- 项目分了哪几层？为什么这么分？
- 异步任务（Celery）解决了什么？哪些活该丢给它？
- 你这个项目最大的短板是什么？（诚实答：测试基本是空的，然后讲你后来补了哪些）

### 诚实路径话术模板（fork 项目怎么讲）

❌ 不要说："这是我独立从 0 开发的项目。"（fork 关系一查就穿帮，AI 岗面试官最爱翻 GitHub）

✅ 推荐这样说：
> "我深入研究了一个开源的 AI 模拟面试项目，吃透了它的架构和 RAG/Agent 设计，并在它基础上做了二次开发——补充了单元测试、给 RAG 加了 rerank 重排、给评分加了缓存。过程中我反推了它的 PRD 和技术规格来理解设计意图。"

这个版本：① 没有穿帮风险；② 突出了你最值钱的能力（读懂大型代码 + 能二开）；③ 每个点都经得起深挖。

### 模拟面试怎么做（用你自己的项目当题源）

真实面试 ~80% 时间在深挖你简历上的项目，纯八股是少数。所以：

1. **题源用你的 `docs/` 和 `specs/`**，而不是泛泛的八股题库
2. 让模型扮演面试官，规则：
   - 一次只问一个问题，根据你的回答**追问**（"那如果…呢？""为什么不用…？"）
   - 答得含糊就继续往深挖，直到挖到你答不上的边界
   - 每轮结束给反馈：哪里答得好、哪里露怯、怎么改进
3. **答不上的地方记下来，回去补**，再来一轮
4. 模型选择：平时多刷几轮用便宜快的（Sonnet），终面冲刺用更强的（Opus）做压力面

模拟面试启动话术示例（喂给模型）：
```
你现在是一位资深 AI 应用开发面试官，正在面试我。
面试材料是我项目的 docs/ 和 specs/ 文档（见上文）。
规则：一次问一个问题，根据我的回答追问，挖到我答不上为止；
每轮给我简短反馈。从我简历上的"岗位匹配 Agent"开始问。
```
