## 通俗类比库（术语翻译器）

用于把专业术语翻译成零基础也能秒懂的大白话。首次出现某术语时，格式：**术语**（大白话 + 类比）。

### Python / FastAPI / 异步生态（重点）

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| FastAPI | 一个写后端接口的框架 | 就像一套半成品厨房，水电煤气都接好了，你只管炒菜（写业务） |
| 路由 / router | 接口的门牌号登记表 | 告诉程序"这个网址该交给哪个函数处理" |
| 依赖注入 / Depends | 外卖模式 | 函数需要数据库连接？别自己造，框架自动送上门 |
| Pydantic / Schema | 表单校验器 | 规定"传进来的数据必须长这样"，不合格直接打回 |
| ORM / SQLAlchemy | 数据库翻译官 | 你写 Python 对象，它翻译成 SQL 去操作数据库 |
| Alembic / migration | 数据库的版本控制 | 像 git，记录每次表结构怎么改的，能升级能回退 |
| async / await | 排队等号但不傻等 | 等接口返回时，程序去干别的活，不卡死 |
| 协程 / coroutine | 会"暂停让位"的函数 | 等 IO 时主动让出，让别的任务先跑 |
| Uvicorn | 给 FastAPI 烧火的灶台 | 真正监听端口、收发请求的服务器进程 |
| SSE / 流式输出 | 边做边端上桌 | 不等整篇答案生成完，AI 写一个字就推一个字给前端 |
| Celery | 后厨的帮工 | 耗时活（发邮件、批量向量化）丢给它慢慢干，主线程不等 |
| Redis | 便利贴 + 排队叫号机 | 既当缓存（贴手边常用数据），又当任务队列的传话筒 |
| Broker | 任务的中转站 | 主程序把活丢进去，Celery 帮工从里面取活干 |

### AI / LLM / RAG / Agent（重点）

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| LLM | 会聊天的超级模型 | 你问它答，但答得对不对要你来兜底 |
| Prompt | 给模型的工单 | 你交代得越清楚，它干得越好 |
| system / user prompt | 岗位说明书 / 当前任务 | system 定人设和规矩，user 是这一次具体要它干嘛 |
| temperature | 模型的"放飞程度" | 调低=稳重保守（适合解析/评分），调高=有创意（适合出题） |
| token | 模型眼里的"字" | 计费和长度都按它算，一个汉字约 1-2 个 |
| Embedding / 向量化 | 把文字翻译成一串坐标 | 意思相近的文字，坐标也相近，方便"按意思找" |
| 向量数据库 / pgvector | 按"意思"检索的仓库 | 不靠关键词匹配，靠坐标远近找最像的内容 |
| cosine_distance | 衡量两段文字像不像 | 两串坐标夹角越小越像，0 最像 |
| RAG | 先查资料再回答 | 模型答题前，先从资料库捞出相关内容塞给它，少瞎编 |
| 召回 / retrieve | 从仓库捞候选 | 按 query 把最相关的 N 条先捞出来 |
| 兜底 / 降级 | 备用方案 | 题库不够就放宽条件再捞，再不够让 AI 现编，保证流程不断 |
| Agent | 会自己规划+调工具的模型 | 不是一问一答，而是"我先查简历→再建画像→再匹配"自己串起来 |
| Tool / 工具 | 给 Agent 的外挂能力 | 每个工具是一个函数，Agent 自己决定啥时候调哪个 |
| Tool docstring | 工具的使用说明书 | Agent 全靠读这段说明，决定要不要调这个工具，必须写准 |
| Tool Calling | 模型"点名"调函数 | 模型输出"我要调 match_positions(画像)"，框架替它执行 |
| AgentExecutor | Agent 的调度循环 | 反复"想→调工具→看结果→再想"，直到给出最终答案 |
| max_iterations | 防止 Agent 鬼打墙 | 最多转几圈，超了强制停，避免死循环烧钱 |
| 结构化输出 | 逼模型吐固定格式 | 要求只返回 JSON，方便程序解析，但模型常不听话要兜底 |

### 后端通用术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| API | 服务员 | 前端点菜、后端做菜、API 传菜 |
| 中间件 | 安检通道 | 请求进来前先过一道检查（鉴权、日志、限流） |
| 数据库 | 仓库 | 存放所有数据的地方 |
| 缓存 | 便利贴 | 常用信息贴手边，不用每次去仓库找 |
| 队列 / MQ | 排队叫号 | 任务太多先排队，一个一个处理 |
| 鉴权 | 查身份证 | 确认"你是谁"以及"你能干什么" |
| 事务 | 要么全做要么全不做 | 银行转账：扣钱和加钱必须同时成功 |
| 幂等 | 按电梯按钮 | 按一次和按十次效果一样，不重复执行 |
| 限流 | 景区限流 | 人太多就不让进，保护系统不被压垮 |

### 架构模式术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| 分层架构 | 流水线 | 每层只干自己的事，按顺序传递（api→service→model） |
| 单例模式 | 全校只有一个校长 | 整个程序只造一个实例反复用（如 LLM 客户端） |
| 工厂模式 | 自动贩卖机 | 告诉它要什么，它自动造一个给你 |
| 策略模式 | 导航选路线 | 同一目的地，可换不同路线（不同召回策略） |
| 观察者模式 | 订阅公众号 | 有新消息自动通知所有订阅者 |
| 依赖注入 | 外卖模式 | 需要什么别自己做，让别人送过来 |

### 前端术语（看管理端/用户端时用）

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| 组件 | 积木块 | 页面由很多积木拼成，每个能单独用 |
| 路由 | 导航地图 | 告诉程序"这个地址去哪个页面" |
| 状态管理 / Pinia | 全局记事本 | 所有页面都能读写的共享数据 |
| Props | 父母给孩子的礼物 | 父组件传给子组件的数据，只看不改 |
| Axios | 跑腿小哥 | 前端用它去后端取数据 |

### 通用编程术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| 变量 | 盒子 | 装数据的容器，可换内容 |
| 函数 | 机器 | 放进原料，出来产品 |
| 回调函数 | 留个电话 | 事情办完了打给你 |
| 继承 | 家族遗传 | 子类自动拥有父类的能力 |
| 递归 | 俄罗斯套娃 | 自己调用自己，一层套一层 |
| 并发 | 一心多用 | 同时处理多件事（不一定真同时，可能快速切换） |

### 用法提醒
- 遇到表里没有的术语，现场造一个保持风格一致的类比即可
- 业务术语（如"采分点""候选人画像"）要用「业务含义 + 技术含义」双重解释
