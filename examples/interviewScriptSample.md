> **来自**：一个 FastAPI + LangChain + pgvector 的 AI 面试项目的真实产物节选。
> **对应产物**：`docs/05-模拟面试问答实战.md`（生成规范见 references/endToEndWorkflow.md §3.7）。
> **仿写时替换**：项目介绍、技术主题、所有数字（0.7 / ×2 / 1024 等）换成目标项目源码里核实过的值；**保留的是格式与说话方式**——第一人称口语、嵌入"数字/坑/取舍"三类细节、每题带追问链和避雷。

## 〇、开场 60 秒项目介绍（必背骨架，可自由发挥血肉）

> 🎙️ "我做的是一个 AI 模拟面试平台，解决的问题是求职者缺少贴岗位、有反馈的面试练习。整个链路是：候选人上传简历，DeepSeek 解析成结构化数据并给简历诊断；然后有一个 LangChain 的 Agent 做岗位匹配，串了五个工具，从岗位模板库里算出最适合的岗位；选定岗位后走题库 RAG 出题——用 pgvector 做语义召回，召回不足有三级降级兜底；面试过程每答一题，评分用 SSE 流式返回，评语像打字机一样出来，分数是锚定题库采分点打的；最后自动生成报告。技术栈是 FastAPI、PostgreSQL 加 pgvector、Redis 加 Celery、LangChain，模型用 DeepSeek 和 DashScope 的 embedding。这个项目我是基于开源深入研究加二次开发的——完整部署过一遍、反推了它的 PRD 和技术规格、补了测试和一些优化，所以里面每个设计我都能讲清楚为什么。"

⚠️ 避雷：开场别超 90 秒、别背名词堆砌；最后一句主动交代"研究+二开"定位，把主动权握在自己手里，比被问出来体面得多。

## Q1 "讲讲你这个 RAG 是怎么做的？"

> 🎙️ "分两条链路说。**入库侧**：管理端批量导题，每道题带参考答案和采分点，落库的时候 embedding 先是 NULL，接口直接返回，然后 Celery 异步调 DashScope 把'题面加参考答案'拼起来做向量化，1024 维，存 pgvector 的 Vector 列。这里有个细节：为什么把答案也拼进去？因为题面经常就一句'解释一下 GIL'，信息量太少，答案里的'线程''互斥锁'这些术语参与匹配之后，简历上写'并发经验'的人也能召回到这道题。
> **出题侧**：拿岗位名加简历技能拼一个 query，向量化之后在 PG 里用 cosine 距离搜，相似度阈值 0.7，召回量是目标题数的两倍——多捞一倍是给后面 AI 挑题留余地。然后按召回数量走三个分支：够就让 AI 从候选里挑题微调，不够就保留题库题、AI 仿着风格补差额，一道都没有就纯 AI 生成兜底，并且每道题打上来源标记，from_bank 还是 ai_fallback，全链路可追溯。"

🔍 **追问链**：
- L1 "为什么阈值是 0.7？" → "调出来的。0.9 基本召回不到东西，永远走纯 AI 兜底，题库白建了；0.5 开始混进弱相关的题，比如搜 Python 后端捞出前端题——脏题进面试是最伤体验的。0.7 是'宁缺毋滥、缺了靠兜底续命'的平衡点。管理端专门做了个检索测试工具，输入模拟 query 看真实的相似度分布，调参就靠它。"
- L2 "两倍召回量的依据？" → "本质是给 AI 一个挑选空间——挑 5 出 10。再大意义不大还费 token，配置里是个 factor，可调。"
- L3 "cosine 距离和相似度什么关系？" → "pgvector 的 cosine_distance 是 0 到 2 越小越像，代码里统一转成 1 减距离当相似度，阈值 0.7 对应距离 0.3，过滤、排序、截断一条 SQL 全做完。"

⚠️ 避雷：只背"五段链路"名词不给数字，一追就穿。**0.7、×2、1024** 这三个数字必须张口就来。

## Q4 "你导入题目之后马上就能被搜到吗？"

> 🎙️ "不能，这是我踩过的坑。批量导入是先落库、embedding 置 NULL、立刻返回，向量化在 Celery 里异步跑。我一开始测试的时候导完题马上开面试，死活召回不到，查了才反应过来检索条件里有 embedding IS NOT NULL，没向量化完的题天然不可见。所以管理端的检索测试按钮不只是调参工具，也是'确认向量化跑完了'的验证工具。另外还有个更隐蔽的坑：题库的 position_tag 和岗位模板的 position_tag 是字符串逻辑关联，没有外键，两边写得不一致——比如下划线和横线——召回静默为空，不报错，全走纯 AI 兜底，特别难发现。"

💡 这题答好了是巨大加分：**说得出"静默失败"的坑，比说得出正确流程更能证明你真跑过**。

## 这个样例展示的范式要点（仿写自查）

1. 逐字稿是**说话**不是文档：有"这里有个细节""诚实说""我一开始…后来才反应过来"这类口语衔接。
2. 每段稿子里埋了可被追问的**钩子**（数字、坑、取舍），追问链就是顺着钩子往下接。
3. 避雷写的是"最容易露馅的答法"，不是泛泛的注意事项。
4. 坑的叙事结构固定：现象（死活召回不到）→ 定位（查了才发现）→ 根因（IS NOT NULL 条件）→ 延伸（检索测试按钮的第二用途）。
